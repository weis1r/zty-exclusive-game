实现思路
- 先把玩法收敛回经典玩法，再逐步压缩局内 HUD 和视觉噪音。
- 三屏结构保持清晰：`首页 -> 单局 -> 结果页`。
- Android 问题统一按“先复现、再截图、再日志、最后修复”处理，不凭印象改。
- 优先修影响真实体验的问题：
  - 页面半屏显示
  - 返回键撤销失效
  - 左上角返回按钮无确认框
  - 牌面太小
  - 残局未自动通关

主要问题与解决方案
- Android 半屏显示
  - 原因: `vh / dvh / 100%` 在 WebView 下不稳定。
  - 方案: 用 `window.innerHeight` 同步 `--app-height`，让 `html / body / #root / app-shell / device-shell` 全链路兜底。

- Android 返回键撤销失效
  - 原因: `WebView` 聚焦时会先消费 `KEYCODE_BACK`。
  - 方案: `MainActivity` 同时接管 `dispatchKeyEvent(KEYCODE_BACK)` 和 `OnBackPressedDispatcher`，统一先调用前端 `window.androidHandleBack()`。

- 左上角返回按钮点击无反应
  - 原因: 前端调用了 `window.confirm(...)`，但 Android WebView 没正确弹出 JS confirm。
  - 方案: 新增 `GameWebChromeClient`，用原生 `AlertDialog` 接管 `onJsConfirm(...)`。

- 单局牌太小
  - 原因: 牌面命中面积偏小，手机上点选吃力。
  - 方案: 单牌点击区域提升约 `50%`，并保持中心点不漂移。

- 剩 3 块未自动通关
  - 原因: 旧逻辑只看棋盘剩余块数，并且要求托盘为空。
  - 方案: 胜利条件改为看“全局未解决块数”，统一计入棋盘、托盘、暂存位，总数 `<= 4` 即判胜。
